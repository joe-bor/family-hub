# Mobile UX Polish Backlog

Items deferred from the mobile calendar redesign (2026-03-18) for a future comprehensive UI/UX polish pass.

## 1. Home Dashboard Redesign

The mobile home screen is a basic 2-column grid of colored module buttons. It works but feels like a prototype. Needs a proper mobile home experience — possibly a dashboard with today's agenda, upcoming chores, quick-add actions, etc.

## 2. Persistent Bottom Navigation

Currently users must return to the Home dashboard to switch modules. A persistent bottom nav (Calendar, Chores, Meals, Lists, Photos) would feel more app-like and reduce friction.

## 3. Visual Identity Refinement

The cream/purple/Nunito foundation is solid. A deeper pass should tighten:
- Consistent spacing scale across all screens
- Better type hierarchy (heading sizes, weight usage, line heights)
- More cohesive member color palette (the 7 colors work but could feel more unified)

During brainstorming, "keep the bones, refine the execution" was chosen over a bigger overhaul — but a more thorough refinement is still wanted.

## 4. Expandable Bottom Sheet Pattern

Full-screen sheets were implemented for event forms (simplest, most reliable). A drag-to-expand bottom sheet (starts half-screen, user pulls up to full) would feel lighter and more modern — worth revisiting once the base mobile experience is solid.

## 5. Sidebar, Settings, and Onboarding Mobile Pass

These screens haven't been specifically optimized for mobile. The sidebar slides in but the settings/onboarding flows could benefit from mobile-native layouts.

## 6. New Interaction Features

- **Notifications** — push or in-app reminders for upcoming events
- **Drag-to-create** — tap and drag on time grid to create an event at that time slot
- **Pinch-to-zoom** — zoom in/out on calendar views to adjust time granularity
